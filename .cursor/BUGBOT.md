# Bugbot Review Rules — PPT PowerKeys

Правила для автоматического ревью PR (Cursor Bugbot). Фокус — на типичных ошибках VSTO/PowerPoint-плагина.

## Всегда проверяй

1. **Утечки COM.** Объекты `Microsoft.Office.Interop.PowerPoint` должны освобождаться; не должно быть цепочек «через две точки», создающих неосвобождаемые временные COM-объекты.
2. **Пустое/малое выделение.** Команды выравнивания/размера/объектов должны корректно обрабатывать случай, когда выделено 0 или 1 объект (no-op, не исключение).
3. **Опорный объект.** Выравнивание/размер считаются относительно последнего выделенного объекта, а не края слайда.
4. **Конфликты шорткатов.** Новые сочетания не должны пересекаться с нативными PowerPoint/Windows и должны оставаться переопределяемыми.
5. **Совместимость.** Не использовать API, недоступные в PowerPoint 2013; код собирается под x86 и x64.
6. **Слои.** Бизнес-логика — в `Commands`/`Core`, не в `UI`.
7. **Настройки.** Никакого хардкода путей/секретов; пользовательские данные — только в `Settings/UserSettings.json`.

## Не флагай
- Отсутствие сборки/тестов в CI на Linux (VSTO собирается только на Windows — это ожидаемо).
