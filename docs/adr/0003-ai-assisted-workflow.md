# 0003. AI-workflow: архитектор + субагенты в Cursor

- **Статус:** Accepted
- **Дата:** 2026-06-26

## Контекст

Команда хочет использовать Cursor так, чтобы:
- субагенты писали код параллельно;
- отдельная «архитекторская» сессия планировала и проверяла PR;
- контекст не терялся между сессиями и агентами.

LLM не хранят память между запусками; в Cursor 2.1 удалены Custom Modes и Memories. Поэтому durable-контекст нужно выносить в файлы репозитория.

## Решение

См. подробное руководство [`docs/cursor-workflow.md`](../cursor-workflow.md). Кратко:

- **Архитектор** — read-only субагент (`.cursor/agents/architect.md`) + Plan Mode; сохраняет планы в репозиторий.
- **Builder** — пишущий субагент (`.cursor/agents/builder.md`), исполняет планы.
- **Reviewer** — read-only субагент + Bugbot для проверки PR.
- **Durable-контекст** — `AGENTS.md`, `.cursor/rules/*.mdc`, ADR, сохранённые планы, `.cursor/BUGBOT.md`.

## Альтернативы

- Custom Modes — удалены из Cursor, недоступны.
- Держать один «вечный» чат живым — ненадёжно, ограничено окном контекста.

## Последствия

- Контекст переживает любую сессию, т.к. лежит в git.
- Требуется поддерживать правила/ADR в актуальном состоянии.
- Сборку/тест VSTO выполняем локально на Windows; cloud-агенты — для правок, рефактора и review.
