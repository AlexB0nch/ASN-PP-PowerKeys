# 0001. Вести Architecture Decision Records

- **Статус:** Accepted
- **Дата:** 2026-06-26

## Контекст

Проект ведётся с активным участием AI-агентов, которые не сохраняют память между сессиями. Нужен durable-источник контекста для архитектурных решений.

## Решение

Вести ADR в `docs/adr/` по единому шаблону. Каждое значимое решение — отдельный пронумерованный файл.

## Альтернативы

- Решения только в PR-описаниях — теряются и плохо ищутся.
- Вики вне репозитория — не версионируется вместе с кодом, недоступна агентам как контекст.

## Последствия

- Решения версионируются вместе с кодом и доступны агентам через `@`-mentions и правила.
- Требуется дисциплина: новое значимое решение → новый ADR.
